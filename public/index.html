Core capabilities

Built the way banks build it

PaySphere implements real fintech patterns used by production financial systems.

📒
Double-Entry Ledger
Every transaction creates two immutable ledger entries — a DEBIT and CREDIT — ensuring your books always balance.
⚡
Atomic Transfers
MongoDB sessions ensure both ledger entries are committed together. Either both succeed or both fail — money never disappears mid-transaction.
🔑
Idempotency Keys
Send the same transfer request multiple times without duplicating charges. Safe retries are built into the protocol.
🛡️
JWT Auth + Blacklisting
Secure cookie-based authentication with automatic token blacklisting on logout to prevent token reuse.
📧
Email Notifications
Automatic email confirmations for registration and every completed transaction via Nodemailer with OAuth2.
📊
Full Audit Trail
Immutable ledger entries mean every balance change is traceable. Query full transaction history sorted by recency.
How it works

From signup to transfer in 4 steps

PaySphere is straightforward by design.

01
Register an account
Create a user with email and password. Get a JWT token back, stored in a secure cookie.
02
Create a wallet
One API call creates your account with ACTIVE status and starts your zero-balance ledger.
03
Deposit funds
Add money by creating a CREDIT ledger entry. Your balance is always computed from ledger entries, never stored directly.
04
Transfer atomically
Send funds to any account. The system validates balance, creates a DEBIT + CREDIT atomically, and emails both parties.
API Reference

Clean, predictable endpoints

All endpoints follow RESTful conventions. Protected routes require your JWT cookie.

POST /api/auth/register Register new user

Request body:

{
  "name": "Jane Doe",
  "email": "jane@example.com",
  "password": "securepass123"
}
POST /api/auth/login Login & get JWT cookie 
{
  "email": "jane@example.com",
  "password": "securepass123"
}
POST /api/auth/logout 🔒 Protected

No body required. Blacklists your token and clears the cookie.

POST /api/account/ 🔒 Protected

No body required. Creates a wallet for the authenticated user.

POST /api/account/deposit 🔒 Protected 
{
  "accountId": "64f1a2b3c4d5e6f7a8b9c0d1",
  "amount": 5000
}
GET /api/account/balance 🔒 Protected

Returns computed balance from ledger (not stored value).

{
  "balance": 4500,
  "currency": "INR"
}
GET /api/account/transactions 🔒 Protected

Returns full transaction history sorted by newest first.

POST /api/transaction/ 🔒 Protected 
{
  "fromAccount": "64f1a2b3c4d5e6f7a8b9c0d1",
  "toAccount":   "64f1a2b3c4d5e6f7a8b9c0d2",
  "amount":      500,
  "idempotencyKey": "unique-uuid-here"
}
